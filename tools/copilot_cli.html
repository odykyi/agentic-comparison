It provides Copilot-powered code explanations and command suggestions right in your terminal. ## Key Differentiator **GitHub Ecosystem Integration**: Copilot CLI is part of the broader GitHub Copilot ecosystem, sharing context with Copilot in VS Code and other editors. ## Prerequisites 1. **GitHub CLI** (`gh`) must be installed 2. **GitHub Copilot** subscription (trial or paid) 3. **Authenticated** with `gh auth login` ## Installation ### Install the extension ```bash gh extension install github/copilot-cli ``` ### Verify installation ```bash gh copilot --help ``` ## Quick Start 1. **Explain code** ```bash gh copilot explain "git reset --hard HEAD~1" ``` 2. **Get command suggestions** ```bash gh copilot suggest "list all running docker containers" ``` ## Commands Reference ### `gh copilot explain` Explain shell commands, code, or concepts: ```bash gh copilot explain "docker ps" ``` ### `gh copilot suggest` Get AI-generated command suggestions: ```bash gh copilot suggest "find large files" ``` ## Pricing ### Copilot Individual - Knowledge bases ## Integration with Copilot Copilot CLI shares your Copilot subscription with: - VS Code extension - JetBrains plugin - Neovim plugin - GitHub Codespaces ## Limitations Compared to other agentic tools: - **No multi-turn conversations** - Single-shot explanations/suggestions - **No file editing** - Read-only, no code modification - **Limited to explain/suggest** - No interactive coding session ## When to Use Copilot CLI ✅ **Great for:** - Understanding complex shell commands - Learning new CLI tools - Quick command suggestions - Explaining code snippets ⚠️ **Not for:** - Interactive coding sessions - Multi-file editing - Project-wide refactoring - Long-running conversations ## Resources - [GitHub Copilot Docs](https://docs.github.com/en/copilot) - [GitHub CLI Manual](https://cli.github.com/manual/) - [Copilot CLI Extension](https://github.com/github/copilot-cli)
